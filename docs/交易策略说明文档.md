## 一、系统架构总览

```
                          实时行情推送
                                │
                  ┌─────────────┼─────────────┐
                  ▼             ▼             ▼
       ┌─────────────────┐  ┌─────────┐  ┌──────────┐
       │ Volume Anomaly  │  │ SEC     │  │ CCASS    │  ← 触发策略
       │ Detector (实时) │  │ Tracker │  │ Tracker  │  (任意一个产生
       └────────┬────────┘  └────┬────┘  └─────┬────┘   BUY/SELL 即触发)
                │                │             │
                └────────────────┼─────────────┘
                                 ▼
                  ┌──────────────────────────────┐
                  │     Strategy Ensemble        │
                  │  (置信度加权 + 弃权机制)      │
                  │                              │
                  │  ┌────────────────────────┐  │
                  │  │ LSTM (90日回看)        │  │ ← 辅助策略
                  │  │ Technical (RSI/MACD等) │  │   (仅投票, 不能触发)
                  │  │ Breakout (通道突破)    │  │
                  │  └────────────────────────┘  │
                  └──────────────┬───────────────┘
                                 ▼
                  ┌──────────────────────────────┐
                  │      Signal Filter           │
                  │  • 置信度过滤(支持触发覆写)   │
                  │  • 港股开盘 10min 噪声过滤   │
                  │  • 反向回补价护栏 (24h)      │
                  │  • 冷却 / 最小持仓 / 反转限频 │
                  └──────────────┬───────────────┘
                                 ▼
                  ┌──────────────────────────────┐
                  │   Order Manager (执行)        │
                  │   ↓ FundGuard (资金 + 杠杆)  │
                  │   ↓ OrderValidator           │
                  │   ↓ CorrelationFilter        │
                  └──────────────┬───────────────┘
                                 ▼
                  ┌──────────────────────────────┐
                  │   Profit / Stop Manager      │
                  │  (固定/部分/追踪止盈, 多档止损)│
                  └──────────────────────────────┘
```

**核心设计原则：触发驱动 + 辅助确认**

- 只有 **触发策略**（Volume Anomaly、SEC、CCASS）产生 BUY/SELL 信号时，系统才会发起交易
- **辅助策略**（LSTM、Technical、Breakout）仅参与投票调节方向和置信度，不能独立触发交易
- 无触发信号时，系统静默等待，不做任何交易
- **市场分工**：SEC 主要服务美股、CCASS 主要服务港股、Volume Anomaly 全市场通用

## 三、策略详解

### 3.1 触发策略一：Volume Anomaly（异常成交量检测）

**角色：** 触发策略（基础权重 30%）

**工作方式：** 实时监听行情推送，持续检测四类异常：

| 异常类型 | 检测逻辑 | 阈值 |
|---------|---------|------|
| **大宗交易 (Block Trade)** | 单次推送成交量 ≥ 日均量的 0.5% | `block_trade_pct: 0.5` |
| **成交量激增 (Volume Surge)** | 累计成交量 ≥ 日均量的 3 倍 | `surge_multiplier: 3.0` |
| **成交量突刺 (Volume Spike)** | 单次成交偏离均值 ≥ 2.5 个标准差 | `spike_std_threshold: 2.5` |
| **量价背离 (Price-Vol Divergence)** | 成交速率 ≥ 均值 2 倍但价格变动 < 0.3% | `divergence_*` |

**方向推断：** 使用 **Tick Rule** — 根据价格变化推断主买/主卖方向：
- 价格上涨 → 主买（BUY 信号）
- 价格下跌 → 主卖（SELL 信号）
- 价格不变 → 沿用上次方向

**多空分歧弃权机制（v4.0 新增）：**
- 当 `|net_direction| < 0.5` 且 `|avg_price_change| < 0.2%` 时，判定多空力量均衡，**返回 None 弃权**，不强行选方向
- 当买卖信号并存且分数比例 > 0.4 时（即买 / 卖力度接近），同样弃权
- 避免在震荡市中被噪声拖入错误方向

**方向一致性检查：** 30 分钟内 ≥ 3 个异常信号时，主方向占比需 **≥ 70%** 才确认方向

**事件驱动：** 检测到异常后立即触发 Ensemble 评估，不等待定期周期

---

### 3.2 触发策略二：SEC 机构跟踪（美股）

**角色：** 触发策略（基础权重 20%）

**工作方式：** 定期扫描 SEC EDGAR 数据库，跟踪知名机构的 13F 和 Form 4 披露。

**跟踪机构：** Berkshire Hathaway、Bridgewater、Renaissance、Citadel、Two Sigma、Point72、BlackRock、JPMorgan、Goldman Sachs、Morgan Stanley、Soros、Appaloosa

**信号生成逻辑：**
- ≥ 2 家机构买入同一标的 → BUY 信号（`min_institutions_buying: 2`）
- 内部人买入 → 置信度加成 +15%
- 无相关数据 → 弃权

**自动加入观察列表：** SEC 13F/Form 4 中出现大额买卖的非池内标的会被自动加入实时订阅列表

**参数：** 扫描间隔 7200s，信号有效期 48h，最小交易金额 $100K

---

### 3.3 触发策略三：CCASS 港股持仓追踪（港股）

**角色：** 触发策略（基础权重 20%，仅对 `.HK` 标的生效）

**工作方式：** 定期抓取 HKEX CCASS（中央结算系统）每日持仓数据，追踪大型参与者（券商/托管行）持仓变化。

**追踪参与者：** 中央结算、JPMorgan、HSBC Custody、Citibank、Standard Chartered 等 10+ 家主要参与者

**信号生成逻辑：**
- 净持仓变化百分比超过 `min_change_pct: 0.5` 阈值 → 产生信号
- 净增持 → BUY；净减持 → SELL
- 美股直接弃权（不参与美股投票）

**HKEX 抓取要点：** 处理 ASP.NET ViewState 状态机制，分离 GET（不带 Content-Type）与 POST 请求

**参数：** 扫描间隔 7200s，对比回看 5 天，追踪前 10 名参与者，信号有效期 24h

---

### 3.4 辅助策略一：LSTM 深度学习

**角色：** 辅助策略（权重 10%）

**模型：** 三层堆叠 LSTM（128→64→32），Dropout 0.3
**输入：** 5 维 — `close, volume, high, low, turnover`，回看 90 个交易日
**信号阈值：** 预测值 > +0.04 → BUY；< -0.04 → SELL；介于之间 → HOLD

---

### 3.5 辅助策略二：Technical 技术指标

**角色：** 辅助策略（权重 10%）
**指标：** RSI、MACD、布林带、SMA20、EMA12，多指标加权评分

---

### 3.6 辅助策略三：Breakout 通道突破（v3.5+ 新增）

**角色：** 辅助策略（权重 10%）

**工作方式：** 检测当前价是否突破近 N 日（默认 20 日）最高/最低价：
- 价格突破 N 日高点 + ATR 余量 → 确认 BUY
- 价格跌破 N 日低点 - ATR 余量 → 确认 SELL
- ATR 用于动态调整突破有效性，避免假突破

**参数：** 回看 20 日，ATR 周期 14，突破余量 0.2%

## 四、Ensemble 组合决策机制

### 4.1 决策流程

```
1. 收集所有策略信号（含弃权判断）
2. 检查触发策略：
   ├── 无任何触发策略产生方向信号 → 强制 HOLD，不交易
   └── 至少一个触发策略产生 BUY/SELL → 进入加权投票
3. 参与策略按归一化权重投票（弃权策略权重重新分配）
4. 计算 buy_strength / sell_strength / hold_strength
5. 通过止盈止损预检（is_near_exit 阻止追买将平仓的标的）
6. 通过 Signal Filter 全套过滤（详见 4.3）
7. 提交 Order Manager 执行
```

### 4.2 权重配置（当前生效值）

| 策略 | 基础权重 | 角色 | 适用市场 |
|------|---------|------|---------|
| volume_anomaly | **30%** | 触发 | 港股 + 美股 |
| sec | 20% | 触发 | 主要美股 |
| ccass | **20%** | 触发 | 仅港股 |
| lstm | 10% | 辅助 | 全部 |
| technical | 10% | 辅助 | 全部 |
| breakout | 10% | 辅助 | 全部 |

**弃权处理：** 策略返回 None 时其权重自动按比例分配给其他参与策略

### 4.3 信号过滤器（v4.0 完整版）

| 过滤维度 | 默认值 | 说明 |
|---------|-------|------|
| 信号去重窗口 | 60 秒 | 同一标的同方向信号 60s 内不重复 |
| 最低置信度 | 0.08 | 低于此阈值的信号被丢弃 |
| **触发策略阈值覆写** | `ccass: 0.05` | CCASS 触发的弱信号单独放宽（v4.0 新增）|
| 港股开盘噪声过滤 | 09:30-09:40 | 港股开盘前 10 分钟 BUY 信号视为集合竞价噪声 |
| 最小持仓时间 | 300 秒 | 反向操作（BUY↔SELL）的最小间隔 |
| 价格变化阈值 | 0.3% | 同一标的价格变化不足时不生成新信号 |
| 每日信号上限 | 10 次/标的 | 防止单日过度交易 |
| 止损后再入场冷却 | 3600 秒 | 止损卖出后 1h 内禁止重新买入 |
| **回补价护栏** | 24h, 0% 溢价 | 24h 内 BUY 价不得高于最近 SELL 价（v4.0 新增）|

**关键设计：**
- **原子检查与记录**：`should_emit_signal` 在锁内完成检查后立即记录信号，消除 TOCTOU 竞态
- **触发策略阈值覆写**：CCASS 单标的信号置信度天然较低（0.05~0.10 区间），全局 0.08 阈值会过滤大部分有效信号；通过 `signal.extra_data.trigger_sources` 识别触发源并应用覆写阈值
- **回补价护栏**：防止 SELL 后短期内追高回补（如 SNDK 案例）；MU 类有利回补（BUY ≤ SELL）则放行
- **止损再入场冷却**：止损退出（`STOP_LOSS`、`TRAILING_STOP`、`EMERGENCY_STOP`、`DAILY_LOSS_LIMIT`）后 1h 内禁止 BUY，止盈退出不触发此冷却

## 五、风险控制

### 5.1 资金守卫（FundGuard）

| 控制维度 | 默认值 | 说明 |
|---------|-------|------|
| 单笔交易限制 | 总权益 5% | `position_pct: 5.0` |
| 总仓位限制 | 总权益 80% | `max_total_position_pct: 80.0` |
| **单标的集中度** | 总权益 20% | `max_single_position_pct: 20.0`（v4.0 新增）|
| 日亏损限制 | 总权益 3% | 触发后暂停当日交易 |
| 最大杠杆倍数 | 2.5x | `max_leverage`，超过拒绝买入 |
| 保证金缓冲预警 | 35% | 触发时记录警告 |
| 保证金缓冲危险 | 25% | 触发时建议减仓 |
| 最小储备金 | $1,000 | `min_reserve` 永不动用 |
| 最小买入豁免 | 1 手 | 首次最小手数买入豁免集中度限制 |

**真实净资产计算：** 优先使用 broker `net_assets`（含未实现盈亏），避免简单 `cash + cost_value` 的口径偏差

**保证金风险监控：** Health Check 任务持续监控 broker 返回的 `risk_level` / `leverage` / `margin_buffer_pct`，提示 STOP_BUYING 或 REDUCE_POSITION 建议

### 5.2 止盈止损

| 类型 | 阈值 | 动作 |
|------|------|------|
| 固定止盈 | +15% | 全部平仓 |
| 部分止盈 | +8% | 平仓 50% |
| 追踪止盈 | 盈利 ≥ +5% 后从高点回撤 | 全部平仓 |
| 固定止损 | -5% | 全部平仓 |
| 紧急止损 | -10% | 立即平仓（含 2% 滑点容忍） |
| 追踪止损 | 从最高价/成本下移 4% | 触发后全部平仓 |
| 单日亏损限制 | 总权益 3% | 暂停交易 |

**追踪止损步进：** `trailing_profit_step: 1.5%`，每盈利 1.5% 上移止损线一次

**止盈止损订单容错（v3.5 加固）：**
- 止盈止损订单失败后保留 `retry_count`，避免无限重试（如 MU.US 47 拒单事件）
- 300s submitted 状态超时后递增 `retry_count` 而非清除
- 美股市场开盘时清除超限重试标记，允许新一日重试

### 5.3 做空控制（仅美股）

| 参数 | 值 | 说明 |
|------|-----|------|
| 做空开关 | 开启 | 仅美股 (`.US` 标的) |
| `allow_sell_to_short` | true | 无多头时 SELL 自动转 SHORT，仅美股 |
| 单只最大空头 | 100 股 | `max_short_position` |
| 总空头敞口 | $50,000 | `max_total_short_exposure` |
| 保证金要求 | 150% | `short_margin_requirement` |

**港股不做空：** 港股 SELL 无持仓时直接拒绝，不会自动转 SHORT

### 5.4 关联性过滤

- 在 `on_signal` 入口对 BUY 信号调用 `portfolio_manager.check_correlation`
- 资产相关性 > 0.7 时拦截，防止同质化集中持仓

## 六、交易执行

### 6.1 交易时段

| 市场 | 时段 | 说明 |
|------|------|------|
| 港股 | 09:30-12:00, 13:00-16:00 (HKT) | 标准时段 |
| 美股盘前 | 04:00-09:30 (ET) | 已启用 |
| 美股正常 | 09:30-16:00 (ET) | 标准时段 |
| 美股盘后 | 16:00-20:00 (ET) | 已启用 |

### 6.2 订单类型

- 默认 **限价单 (LO)**
- 买价上浮 0.3%，卖价下浮 0.3%（`price_adjust_rate`）
- 紧急止损单额外容忍 2% 滑点
- 港股按交易所 Tick Size 自动归整

### 6.3 手数处理

- 港股严格按手数下单：API 动态查询 + 配置 `lot_sizes` 回退 + 硬编码兜底
- 美股最小单位 1 股
- 数量不足一手时自动调整到最近的手数倍数

### 6.4 订单状态闭环（v3.5 加固）

- 订单提交后写入 `logs/orders.csv`，订单状态变更时**原子更新 CSV**（`tempfile + os.replace`）
- 解决了历史"全部 NotReported"的 bug
- `_track_orders` 唯一负责状态更新；`_save_order` 不再被重复调用

### 6.5 止盈止损订单特权

- `profit_stop_` 开头的策略名自动跳过买入侧风控（卖出侧仅跳过仓位检查，仍走券商风控）
- 确保止损单不会被日常风控限制阻断

## 九、关键设计决策

1. **触发驱动而非定时交易**：只有大宗交易、SEC 披露或 CCASS 持仓变化时才发起交易，避免噪音
2. **市场分工的触发策略组合**：SEC 主要服务美股、CCASS 主要服务港股、Volume Anomaly 全市场通用，避免单一信号源失效
3. **弃权机制**：无数据的策略自动弃权，权重重新分配，避免"无数据投 HOLD"拉低信号质量
4. **多空分歧 = HOLD**：Volume Anomaly 在多空力量均衡时主动弃权，不强行选方向
5. **港股开盘噪声过滤**：09:30-09:40 期间集合竞价放量被识别为噪声，BUY 信号被抑制
6. **触发策略阈值覆写**：CCASS 类弱信号策略可单独配置更低的置信度阈值，避免被全局阈值过滤
7. **回补价护栏**：防止 SELL 后短期内高位 BUY，区分有利回补（MU 类）与追涨回补（SNDK 类）
8. **止损再入场冷却**：止损退出后 1h 内禁止同标的重新买入，打破"止损→买入→止损"循环
9. **保证金主动监控**：基于 broker `risk_level` / `leverage` 实时拦截高风险买入，避免被动平仓
10. **再平衡已禁用**：投资组合管理仅更新状态，不自动再平衡，避免与触发驱动策略矛盾
11. **原子信号记录**：检查与记录在同一锁内完成，消除"卖出后立即买入"的 TOCTOU 竞态
12. **CSV 原子更新**：订单状态使用 tempfile + os.replace 原子更新，避免数据损坏
