## 一、系统架构总览

```
实时行情推送 ──┐
               │
               ▼
     ┌──────────────────┐
     │ Volume Anomaly   │──── 检测到大宗交易 ────┐
     │ Detector (实时)   │                       │
     └──────────────────┘                       ▼
                                     ┌────────────────────┐
     ┌──────────────────┐            │                    │
     │ SEC Institutional│──── 披露 ──▶│  Strategy Ensemble │
     │ Tracker (定期扫描)│            │  (触发+辅助投票)    │
     └──────────────────┘            │                    │
                                     │  ┌──────────────┐  │
     ┌──────────────────┐            │  │ LSTM 预测     │  │
     │ LSTM Model       │──── 辅助 ──▶│  │ Technical指标 │  │
     │ (90日回看)        │            │  └──────────────┘  │
     └──────────────────┘            └────────┬───────────┘
                                              │
     ┌──────────────────┐                     ▼
     │ Technical        │            ┌────────────────────┐
     │ Strategy         │──── 辅助 ──▶│  Signal Filter     │
     └──────────────────┘            │  (去重+冷却+一致性)  │
                                     └────────┬───────────┘
                                              │
                                              ▼
                                     ┌────────────────────┐
                                     │  Order Manager     │
                                     │  (风控 + 执行)      │
                                     └────────┬───────────┘
                                              │
                                              ▼
                                     ┌────────────────────┐
                                     │  Profit/Stop Mgr   │
                                     │  (止盈止损监控)      │
                                     └────────────────────┘
```

**核心设计原则：触发驱动 + 辅助确认**

- 只有**触发策略**（Volume Anomaly、SEC）产生 BUY/SELL 信号时，系统才会发起交易
- **辅助策略**（LSTM、Technical）仅参与投票调节方向和置信度，不能独立触发交易
- 无触发信号时，系统静默等待，不做任何交易

## 三、策略详解

### 3.1 触发策略一：Volume Anomaly（异常成交量检测）

**角色：** 触发策略（权重 40%）

**工作方式：** 实时监听行情推送，持续检测四类异常：

| 异常类型 | 检测逻辑 | 阈值 |
|---------|---------|------|
| **大宗交易 (Block Trade)** | 单次推送成交量 ≥ 日均量的 0.5% | `block_trade_pct: 0.5` |
| **成交量激增 (Volume Surge)** | 累计成交量 ≥ 日均量的 3 倍 | `surge_multiplier: 3.0` |
| **成交量突刺 (Volume Spike)** | 单次成交偏离均值 ≥ 2.5 个标准差 | `spike_std_threshold: 2.5` |
| **量价背离 (Price-Vol Divergence)** | 成交速率 ≥ 均值 2 倍但价格变动 < 0.3% | `divergence_*` |

**方向推断：** 使用 **Tick Rule** — 根据价格变化推断主买/主卖方向：
- 价格上涨 → 主买（BUY 信号）
- 价格下跌 → 主卖（SELL 信号）
- 价格不变 → 沿用上次方向

**方向一致性检查：** 当 30 分钟内存在 ≥ 3 个异常信号时，主方向信号占比需 **≥ 70%** 才确认方向。方向频繁翻转（如 buy/sell 交替出现）的信号被视为噪声，策略弃权不参与投票。方向一致性越高，最终置信度也越高（乘以一致性系数）。

**事件驱动：** 检测到异常后立即触发 Ensemble 评估，不等待定期周期。

**参数配置：**
- 基线计算：30 日历史均量，算术平均，过滤零成交日，至少需 5 日有效数据 (`lookback_days: 30`)
- 最小置信度：0.5 (`min_anomaly_confidence: 0.5`)
- 信号有效期：30 分钟 (`signal_expiry_minutes: 30`)
- 检查间隔：60 秒 (`check_interval: 60`)

---

### 3.2 触发策略二：SEC 机构跟踪

**角色：** 触发策略（权重 20%）

**工作方式：** 定期扫描 SEC EDGAR 数据库，跟踪知名机构的 13F 和 Form 4 披露。

**跟踪机构：**
巴菲特(Berkshire)、达里奥(Bridgewater)、文艺复兴科技、城堡投资、Two Sigma、科恩(Point72)、贝莱德、摩根大通、高盛、摩根士丹利、索罗斯基金、泰珀(Appaloosa)

**信号生成逻辑：**
- ≥ 2 家机构买入同一标的 → 产生 BUY 信号（`min_institutions_buying: 2`）
- 内部人买入 → 置信度加成 +15%（`insider_buy_confidence_boost: 0.15`）
- 无相关数据时 → 弃权（不投 HOLD，直接退出投票）

**参数配置：**
- 扫描间隔：2 小时 (`scan_interval: 7200`)
- 信号有效期：48 小时 (`signal_expiry_hours: 48`)
- 最小交易金额：$100,000 (`min_transaction_value: 100000`)

---

### 3.3 辅助策略一：LSTM 深度学习

**角色：** 辅助策略（权重 20%），仅确认方向，不能触发交易

**模型结构：** 三层堆叠 LSTM（128→64→32 单元），Dropout 0.3

**输入特征：** 5 维 — `close, volume, high, low, turnover`

**回看窗口：** 90 个交易日 (`lookback_period: 90`)

**信号逻辑：**
- 预测值 > +0.04 → BUY（`buy_threshold: 0.04`）
- 预测值 < -0.04 → SELL（`sell_threshold: -0.04`）
- 介于 ±0.04 之间 → HOLD
- 置信度 = |预测值|

**训练数据：** 全量标的联合训练

---

### 3.4 辅助策略二：Technical 技术指标

**角色：** 辅助策略（权重 20%），仅确认方向，不能触发交易

**使用指标：** RSI、MACD、布林带、SMA20、EMA12 等

**信号逻辑：** 基于多指标综合评分生成 BUY/SELL/HOLD

## 四、Ensemble 组合决策机制

### 4.1 决策流程

```
1. 收集所有策略信号（含弃权判断）
2. 检查是否有触发策略产生 BUY/SELL
   ├── 无触发 → 强制 HOLD，不交易
   └── 有触发 → 继续加权投票
3. 参与策略按归一化权重投票
4. 买入强度 vs 卖出强度 vs 持有强度 → 决定方向
5. 通过信号过滤器（去重 + 止损冷却 + 置信度检查）
6. 生成最终交易信号
```

### 4.2 权重配置

| 策略 | 基础权重 | 角色 |
|------|---------|------|
| volume_anomaly | **40%** | 触发 |
| sec | 20% | 触发 |
| lstm | 20% | 辅助 |
| technical | 20% | 辅助 |

**弃权处理：** 如果某策略弃权（返回 None），其权重自动按比例分配给其他参与策略。例如 SEC 弃权时，volume_anomaly 实际权重提升至 ~67%。

### 4.3 信号过滤器

| 参数 | 值 | 说明 |
|------|-----|------|
| 信号去重窗口 | **60 秒** | 同一标的同方向信号 60 秒内不重复 |
| 止损后再入场冷却 | **15 分钟** | 止损卖出后 15 分钟内禁止同一标的重新买入 |
| 最低置信度 | **0.08** | 低于此阈值的信号被丢弃 |
| 每日信号上限 | 10 次/标的 | 防止单日过度交易 |
| 价格变化阈值 | 1% | 同一标的价格变化 < 1% 时不生成新信号 |

**止损再入场冷却：** 当止盈止损管理器执行止损退出（`STOP_LOSS`、`EMERGENCY_STOP`、`TRAILING_STOP`、`DAILY_LOSS_LIMIT`）后，自动进入 15 分钟冷却期。冷却期内该标的的 BUY 信号会被过滤，防止"止损→买入→止损"循环。止盈退出不触发此冷却。

## 五、风险控制

### 5.1 订单级风控

| 控制项 | 参数 | 说明 |
|--------|------|------|
| 单笔交易限制 | 总权益的 **5%** | `position_pct: 5.0` |
| 最大持仓数量 | 2000 股 | `max_position_size: 2000` |
| 最小交易金额 | $200 | `min_trade_value: 200` |
| 最大挂单数 | 8 | `max_pending_orders: 8` |
| 每日订单上限 | 不限 | `max_daily_orders: 999999` |
| 资金检查 | **优先使用 Buy Power** | 避免因做空导致现金为负而误拒订单 |
| 最小储备金 | $1,000 | `min_reserve: 1000` |

### 5.2 止盈止损

| 类型 | 阈值 | 动作 |
|------|------|------|
| 固定止盈 | **+15%** | 全部平仓 |
| 部分止盈 | **+8%** | 平仓 50% |
| 追踪止盈 | 盈利 ≥ +5% 后从高点回调 1% | 全部平仓 |
| 固定止损 | **-5%** | 全部平仓 |
| 紧急止损 | **-10%** | 立即全部平仓（含 2% 滑点容忍） |
| 追踪止损 | 从成本价/最高价下移 3% | 触发后全部平仓 |
| 单日亏损限制 | 总权益的 **3%** | 超过则暂停交易 |

**止损回调机制：** 止损触发后，通知信号过滤器进入 30 分钟再入场冷却期（见 4.3 节），防止亏损循环。

### 5.3 做空控制

| 参数 | 值 | 说明 |
|------|-----|------|
| 做空开关 | 开启 | 支持美股做空 |
| 单只最大空头 | 100 股 | `max_short_position: 100` |
| 总空头敞口 | $50,000 | `max_total_short_exposure: 50000` |
| 保证金要求 | 150% | `short_margin_requirement: 1.5` |
| SELL→SHORT 自动转换 | 是 | 无多头持仓时自动转为开空 |
| 做空黑名单 | 动态 | 交易所拒绝后自动加入黑名单（如 388.HK） |

## 六、交易执行

### 6.1 交易时段

| 市场 | 时段 | 说明 |
|------|------|------|
| 港股 | 09:30-12:00, 13:00-16:00 (HKT) | 标准交易时段 |
| 美股盘前 | 04:00-09:30 (ET) | 已启用 |
| 美股正常 | 09:30-16:00 (ET) | 标准交易时段 |
| 美股盘后 | 16:00-20:00 (ET) | 已启用 |

### 6.2 订单类型

- 默认使用 **限价单 (LO)**
- 买入价格上浮 0.3%，卖出价格下浮 0.3%（`price_adjust_rate`）
- 紧急止损单额外容忍 2% 滑点以确保成交

### 6.3 手数处理

- 港股严格按手数下单（API 动态查询 + 硬编码回退）
- 美股最小单位 1 股
- 数量不足一手时自动调整到最近的手数倍数

### 6.4 止盈止损订单特权

- `profit_stop_` 开头的策略名自动跳过风控检查（买入和卖出方向均适用）
- 确保止损单不会被日常风控限制阻断

### 6.5 止盈止损订单状态管理

- 提交止盈止损订单后标记为 `submitted`，阻止重复提交
- 订单到达终态（成交/拒绝/取消）后通过回调更新状态
- 成交 → 清除 pending 状态；拒绝/取消 → 允许重试
- 超时保护：submitted 状态超过 300 秒自动清除，防止永久阻塞

## 九、关键设计决策

1. **触发驱动而非定时交易：** 只有检测到大宗交易或机构披露时才发起交易，避免噪音交易
2. **弃权机制：** 无数据的策略自动弃权，权重重新分配，避免"无数据投 HOLD"拉低信号质量
3. **方向一致性过滤：** Volume Anomaly 要求 30 分钟内 ≥ 70% 同方向信号才确认方向，过滤高频噪声
4. **止损再入场冷却：** 止损退出后 15 分钟内禁止同标的重新买入，打破"止损→买入→止损"循环
5. **Buy Power 优先：** 资金检查使用券商提供的购买力而非简单现金累加，支持保证金交易
6. **60 秒去重而非限流：** 冷却期仅防止同一事件重复触发，不阻碍连续的独立信号
7. **再平衡已禁用：** 投资组合管理仅更新状态，不自动执行再平衡下单，避免与触发驱动策略矛盾
